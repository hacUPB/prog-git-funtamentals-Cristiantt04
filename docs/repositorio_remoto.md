Creación de un repositorio remoto en GitHub y sincronización con uno local
Un repositorio remoto en GitHub es un espacio en la nube donde se almacena el código del proyecto, permitiendo respaldo y colaboración.


Primero, desde la página web de GitHub, se crea un nuevo repositorio indicando su nombre, descripción y visibilidad (público o privado). Cuando el proyecto ya existe de forma local, es recomendable no inicializar el repositorio remoto con archivos adicionales para evitar conflictos.


Una vez creado, GitHub proporciona una dirección (URL) del repositorio remoto. Esta dirección se utiliza para vincular el repositorio local con el remoto, estableciendo una conexión permanente entre ambos.


Después de enlazarlos, se envía el contenido del repositorio local al remoto. Esto publica los archivos y el historial de cambios en GitHub, dejando ambos repositorios sincronizados.


A partir de ese momento, el flujo de trabajo consiste en realizar cambios localmente, guardarlos y sincronizarlos, manteniendo el proyecto actualizado tanto en el computador como en GitHub.
